About Archivematica

Archivematica is a suite of open-source tools in an open source wrapper that helps users carry out OAIS-compliant digital preservation tasks on their content.

Archivematica's main function is to create Archival Information Packages (AIPs) for long-term storage.

In the middle

We think of Archivematica as part of an ecosystem of digital preservation tools. Just as it fulfils a part of the OAIS requirements, Archivematica fulfils part - the AIP-creating part - of the lifecycle of your content.

End to end

Two common ways to connect Archivematica to a repository:

  • Archivematica first - content is first processed through Archivematica and DIPs are passed to the repository (i.e. AtoM, ArchivesSpace)
  • Archivematica last - content is first added to the repository and then passed to Archivematica for preservation processing (i.e. Islandora, DSpace)

How we develop

Proof-of-concept projects - done as R&D projects, code may not be stable enough to merge into release so held in artefactual-labs repo

Sponsored features/feature enhancements - more formal projects, usually include a 10% community fee that pays for merging new code into next release

Hydra: York/Hull pilot /1

Filling the Digital Preservation Gap, a Jisc Research Data Spring project.

The University of York and the University of Hull had established Hydra-based institutional repositories, but no digital preservation capacity.

Goal: establish an Archivematica-first workflow for their institutional repositories.

Hydra: York/Hull pilot /2

Automated on-demand DIP generation

  • Problem: on-demand DIP generation only available through a manual process
  • Solution: extend AIP Re-ingest workflow by adding automation steps. Hydra can now programmatically find out whether a DIP exists for a given AIP by requesting information via API, and can then trigger an AIP re-ingest to start the DIP creation process.

Hydra: York/Hull pilot /3

Improved METS parsing

  • Problem: Archivematica METS.xml files are huge and unwieldy
  • Solution: develop METS Reader/Writer, a library that provides an API for working with Archivematica METS files. METS Reader/Writer converts METS.xml to JSON for more efficient processing, allowing the data to be reused in Hydra.

Hydra: York/Hull pilot /4

Proof-of-concept project "enhanced the facilities of Archivematica so that the DIP can be more easily used as the basis for a repository object which will be ingested for discovery and delivery."

- Filling the Digital Preservation Gap. Phase Three report

ArchivesSpace: appraisal tab /1

Sponsor: Bentley Historical Library, University of Michigan

"A place between Transfer and Ingest where archivists can make archival appraisal decisions with material from Transfer, and structure an arrangement for the SIP before starting Ingest processes."

- Archivematica wiki > Appraisal Arrangement tab

ArchivesSpace: appraisal tab /2

Goal: improve ArchivesSpace matching GUI

New ArchivesSpace matching GUI allows users to:

  • Match digital objects to their AS resource
  • Add new child records to an existing AS resource (i.e. create a hierarchical arrangement based on the contents of the transfer)
  • Add and edit descriptive metadata and pass it to AS
  • Add and edit rights metadata and pass it to AS

ArchivesSpace: PREMIS rights /1

Sponsor: Rockefeller Archive Center

"Allow users to apply PREMIS rights to individual objects by including a CSV file with the transfer, rather than having all of the objects in the transfer inherit rights applied to the SIP via the GUI."

- Archivematica wiki > Object-level PREMIS rights import

ArchivesSpace: PREMIS rights /2

Goal: indicate per-object rights statements, rather than applying one rights statement to the whole transfer, and then pass information to ArchivesSpace

New CSV template allows users to:

  • Pass detailed, per-object PREMIS rights information to the dmdSec of the object
  • Pass rights information to the ArchivesSpace resource using the matching GUI or by including matching instructions within the transfer

Growth by integration

Integrating with repository platforms is a key way that Archivematica continues to grow. By extending the application to play nice with other systems, it is enriched for everyone.

Integrating to improve

Making sure that integration work emphasises flexibility and generic-ness of both data inputs and outputs (as much as we can) is important for ensuring that growth is both sustainable and useful.
